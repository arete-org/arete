Deployment guide & technical overview

Full documentation is on the way. This preview gives you enough structure to bring a test deployment online until the canonical guide is published.

1. Prerequisites

  • Docker Desktop or a compatible container runtime.
  • Node.js 20+ if you prefer to run services directly.
  • A Discord application with bot permissions and the message content intent enabled.

2. Configure secrets

Duplicate .env.example to .env in the repository root. Supply your OpenAI credentials, Discord bot token, and any other provider keys noted in the sample file.

3. Start the stack

Run docker compose up --build from the repository root. Docker will assemble the backend, web client, and supporting workers.

Prefer manual control? Each package has an npm run dev script so you can iterate locally while proxying through Vite.

4. Authorise your bot

Use the Discord OAuth URL generator with scopes bot and applications.commands. Grant the permissions you are comfortable with (the sample configuration recommends Send Messages, Read Message History, and Use Slash Commands).

5. Configure values

Inside Discord, run /configure and walk through the prompts that describe your community norms. Daneel stores these guardrails securely and honours them in every conversation.

6. Stay in touch

The roadmap is evolving. Watch the repository or join the development Discord server to help guide the upcoming release notes.
